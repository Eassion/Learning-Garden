HTTP 是明文传输，HTTPS 是在 HTTP 和 TCP 之间加了一层 TLS/SSL 加密层，用来保证数据安全。



HTTP 和 HTTPS 的主要区别在于安全性。HTTP 是明文传输，数据在网络中传输时可能被窃听、篡改或者伪造；HTTPS 是在 HTTP 和 TCP 之间加入了 TLS/SSL 安全层，对传输内容进行加密，并通过数字证书验证服务器身份，同时保证数据完整性。

从端口上看，HTTP 默认使用 80 端口，HTTPS 默认使用 443 端口。从连接过程上看，HTTP 在 TCP 三次握手之后就可以直接发送请求；而 HTTPS 在 TCP 连接建立之后，还需要进行 TLS 握手，用来协商加密算法、验证服务器证书，并生成会话密钥。之后 HTTP 请求和响应都会使用这个会话密钥进行加密传输。

所以 HTTPS 相比 HTTP 更安全，但也会带来一定的 TLS 握手和加解密开销。不过现在通过 TLS 1.3、会话复用、HTTP/2 等优化，性能影响已经比较小。实际项目中，登录、支付、接口请求等场景基本都会使用 HTTPS。
